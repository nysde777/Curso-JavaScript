Tema 1: Fundamentos

Historia breve de JS y su rol en la web moderna

JavaScript fue creado en 1995 por Brendan Eich. Inicialmente llamado Mocha y luego LiveScript, finalmente se renombró a JavaScript.

Hoy en día, JavaScript es el lenguaje de programación más importante para el desarrollo web, permitiendo interactividad y dinamismo en las aplicaciones.

// JavaScript ejecutándose en navegadores y servidores (Node.js)
Tema 1: Fundamentos

Sintaxis básica: variables (let, const, var), tipos de datos

Las variables son contenedores para almacenar datos. JavaScript ofrece tres formas de declarar variables:

const nombre = "Juan"; // No se puede reasignar
let edad = 25; // Se puede reasignar
var ciudad = "Madrid"; // Antigua forma, evitar usar

Tipos de datos: string, number, boolean, object, undefined, null, symbol, bigint

Tema 1: Fundamentos

Operadores: aritméticos, lógicos, relacionales, ternario

Aritméticos: +, -, *, /, %, **

Lógicos: &&, ||, !

Relacionales: ==, ===, !=, !==, <, >, <=, >=

const resultado = edad > 18 ? "Mayor de edad" : "Menor de edad";
Tema 1: Fundamentos

Estructuras de control: if, switch, for, while, do...while

Las estructuras de control permiten controlar el flujo de ejecución del programa.

if (condicion) { /* ... */ }
switch (valor) { case 1: /* ... */ }
for (let i = 0; i < 10; i++) { /* ... */ }
while (condicion) { /* ... */ }
Tema 1: Fundamentos

Funciones: declarativas, expresiones, arrow functions

Las funciones son bloques de código reutilizable.

function saludar() { } // Declarativa
const saludar = function() { } // Expresión
const saludar = () => { } // Arrow function
Tema 1: Fundamentos

Scope y hoisting

Scope: Es el contexto donde las variables están disponibles. Puede ser global, de función o de bloque.

Hoisting: Es el comportamiento de mover declaraciones al inicio del scope.

// Las funciones se elevan completamente
// Las variables (let/const) se elevan pero sin inicializar

Template literals

Forma moderna de crear cadenas de texto con interpolación y múltiples líneas.

const nombre = "Juan";
const mensaje = `Hola, ${nombre}!`;
const texto = `
Primera línea

Tema 2: Objetos y Arrays

Objetos: creación, propiedades, métodos

Los objetos son colecciones de pares clave-valor.

const persona = {
  nombre: "Juan",
  edad: 25,
  saludar() { return "Hola"; }
};
Tema 2: Objetos y Arrays

Arrays: creación, métodos modernos (map, filter, reduce, find, some, every, flatMap)

Los arrays son listas ordenadas de valores.

const nums = [1, 2, 3, 4, 5];
nums.map(n => n * 2); // [2, 4, 6, 8, 10]
nums.filter(n => n > 2); // [3, 4, 5]
nums.reduce((sum, n) => sum + n, 0); // 15
Tema 2: Objetos y Arrays

Desestructuración de objetos y arrays

Forma moderna de extraer valores de objetos y arrays.

// Objetos
const { nombre, edad } = persona;

// Arrays
const [primero, segundo] = [10, 20];
Tema 2: Objetos y Arrays

Spread/rest operators

El operador ... (spread/rest) permite expandir o recopilar elementos.

const arr1 = [1, 2];
const arr2 = [...arr1, 3, 4]; // [1, 2, 3, 4]

function suma(...numeros) { // rest
  return numeros.reduce((a, b) => a + b, 0);
}
Tema 2: Objetos y Arrays

JSON: parse y stringify

JSON es un formato de intercambio de datos basado en texto.

const obj = { nombre: "Juan", edad: 25 };
const json = JSON.stringify(obj);
const objeto = JSON.parse(json);
Tema 3: Funciones Avanzadas

Closures

Un closure es una función que tiene acceso a las variables de su función padre, incluso después de que la función padre haya terminado.

function contador() {
  let count = 0;
  return () => ++count; // Acceso a count
}
Tema 3: Funciones Avanzadas

Funciones de orden superior

Son funciones que reciben otras funciones como parámetros o las retornan.

const multiplicar = (n) => (m) => n * m;
const doble = multiplicar(2);
console.log(doble(5)); // 10
Tema 3: Funciones Avanzadas

Callback functions

Son funciones pasadas como argumento a otra función para ser ejecutadas posteriormente.

function obtenerDatos(callback) {
  setTimeout(() => {
    callback("Datos obtenidos");
  }, 1000);
}
Tema 3: Funciones Avanzadas

IIFE (Immediately Invoked Function Expressions)

Son funciones que se invocan inmediatamente después de ser definidas.

(function() {
  let variable = "privada";
  console.log(variable);
})();
Tema 3: Funciones Avanzadas

Currying y composición básica

Currying: Convertir una función que toma múltiples argumentos en varias funciones que toman un argumento.

const suma = (a) => (b) => a + b;
const suma5 = suma(5);
console.log(suma5(3)); // 8
Tema 4: Programación Orientada a Objetos

Clases y constructor

Las clases son plantillas para crear objetos con propiedades y métodos.

class Persona {
  constructor(nombre, edad) {
    this.nombre = nombre;
    this.edad = edad;
  }
}
Tema 4: Programación Orientada a Objetos

Métodos y propiedades

Los métodos son funciones dentro de una clase. Las propiedades son variables.

class Persona {
  saludar() {
    return `Hola, soy ${this.nombre}`;
  }
}
Tema 4: Programación Orientada a Objetos

Herencia (extends y super)

Permite que una clase herede propiedades y métodos de otra.

class Empleado extends Persona {
  constructor(nombre, edad, salario) {
    super(nombre, edad);
    this.salario = salario;
  }
}
Tema 4: Programación Orientada a Objetos

Encapsulamiento: getters y setters

Permite controlar el acceso a las propiedades de una clase.

class Persona {
  get edad() { return this._edad; }
  set edad(valor) { this._edad = valor; }
}
Tema 4: Programación Orientada a Objetos

Prototipos y prototypal inheritance (básico y moderno)

JavaScript utiliza herencia basada en prototipos. Cada objeto tiene un prototipo del cual hereda propiedades.

Object.create() // Crear objeto con prototipo específico
Object.getPrototypeOf() // Obtener prototipo
constructor.prototype // Acceso al prototipo
Tema 5: Programación Funcional

Conceptos de inmutabilidad y funciones puras

Inmutabilidad: No modificar datos existentes, crear nuevos datos en su lugar.

Funciones puras: Siempre devuelven el mismo resultado con los mismos inputs, sin efectos secundarios.

// Pura
const suma = (a, b) => a + b;
Tema 5: Programación Funcional

Uso de map, reduce, filter

Métodos clave para la programación funcional.

const nums = [1, 2, 3, 4, 5];
nums.filter(n => n > 2).map(n => n * 2).reduce((a, b) => a + b, 0);
Tema 5: Programación Funcional

Funciones anónimas y arrow functions

Funciones sin nombre que se usan frecuentemente en callbacks y métodos de array.

const numeros = [1, 2, 3];
numeros.forEach((n) => console.log(n));
Tema 5: Programación Funcional

Closures aplicados a funciones funcionales

Los closures permiten crear funciones que "recuerdan" el contexto en el que fueron creadas.

const crearSumador = (inicial) => {
  return (n) => inicial + n;
};
Tema 6: Módulos y Scope

import y export (ES Modules)

Forma moderna de organizar código en módulos reutilizables.

// archivo.js
export const variable = "valor";
export function funcion() { }

// otro.js
import { variable, funcion } from './archivo.js';
Tema 6: Módulos y Scope

Scope global vs local

Global: Variables accesibles desde cualquier lugar.

Local: Variables accesibles solo en su función o bloque.

let global = "valor"; // Global
function local() {
  let local = "valor"; // Local
}
Tema 6: Módulos y Scope

Variables privadas con # en clases

El símbolo # permite crear propiedades realmente privadas en clases.

class Persona {
  #edad = 0; // Privada
  setEdad(valor) { this.#edad = valor; }
}
Tema 7: Asincronía

Callbacks y problemas de callback hell

Los callbacks pueden llevar a código anidado y difícil de leer (callback hell).

function operacion(cb) {
  setTimeout(() => cb("resultado"), 1000);
}
Tema 7: Asincronía

Promises: creación, encadenamiento (then, catch, finally)

Las promesas representan un valor que puede estar disponible ahora, en el futuro o nunca.

new Promise((resolve, reject) => {
  resolve("éxito");
}).then(res => console.log(res))
 .catch(err => console.error(err));
Tema 7: Asincronía

Async/await

Forma moderna y legible de trabajar con promesas.

async function obtenerDatos() {
  try {
    const res = await fetch('/api');
    return await res.json();
  } catch(e) { console.error(e); }
}
Tema 7: Asincronía

Manejo de errores en asincronía (try/catch)

Los bloques try/catch permiten manejar errores en código asincrónico.

try {
  const datos = await fetch('/api');
} catch(error) {
  console.error("Error:", error);
}
Tema 7: Asincronía

Fetch API y manejo de datos externos

Fetch permite hacer peticiones HTTP desde el navegador.

fetch('/api/datos')
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));
Tema 8: Eventos y DOM

Manipulación del DOM (querySelector, getElementById, appendChild, etc.)

El DOM (Document Object Model) permite interactuar con elementos HTML desde JavaScript.

const elemento = document.querySelector('.clase');
elemento.appendChild(nuevoElemento);
elemento.textContent = "Nuevo texto";
Tema 8: Eventos y DOM

Eventos: click, input, change, submit, keydown

Los eventos permiten reaccionar a las acciones del usuario.

elemento.addEventListener('click', (e) => {
  console.log('Click!');
});

input.addEventListener('input', (e) => {
  console.log(e.target.value);
});
Tema 8: Eventos y DOM

Event delegation

Técnica de escuchar eventos en un elemento padre para elementos hijos.

contenedor.addEventListener('click', (e) => {
  if (e.target.matches('button')) {
    console.log('Botón clickeado');
  }
});
Tema 8: Eventos y DOM

Formulario y validación básica

Validar y procesar datos de formularios.

formulario.addEventListener('submit', (e) => {
  e.preventDefault();
  const valor = input.value;
  if (valor.length > 0) { /* procesar */ }
});
Tema 8: Eventos y DOM

Event loop, call stack y task queue

Conceptos fundamentales de cómo JavaScript ejecuta código asincrónico.

Call Stack: Pila de funciones en ejecución.

Task Queue: Cola de tareas asincrónicas pendientes.

Event Loop: Controla la ejecución de tareas.

// Bueno - sin efecto secundario
